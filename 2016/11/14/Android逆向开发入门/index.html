【学无止境】Android逆向开发入门

  • Android

前期准备

  1. 配置好JDK和Android SDK。
  2. 下载ApkTool，用来反编译apk和重新打包成apk。
  3. 下载Auto-sign，并且解压，这是用来对Apk进行重新签名的。
  4. 找到需要逆向开发的App（这个按个人需求定啦），为了实验，我就在网上随便找了个计步器App。 a

到目前为止，我们已经把逆向开发的环境准备好了。

开发实践

现在我们要对这个apk注入一个按钮，点击这个按钮可以弹出一个Toast。

  1. 首先需要反编译Apk，把Apk和ApkTool放到同一目录，输入
    java -jar apktool.jar d com.pedometers_220545.apk后回车，即可在当前目录下，生成com.pedometers_220545 目录。

    注：apktool用法
  2. 打开com.pedometers_220545目录，可以看到
    • –AndroidManifest.xml: 就是Android常见的文件，反编译后也可以直接阅读，里面声明了Activity等。如果你想增加一个Activity或者修改包名等，就应该对它进行修改。
    • –apktool.yml: 属于反编译配置文件，重新打包时必须有，一般不用修改。
    • –original: 里面有重新打包的AndroidManifest.xml和签名文件，一般不用修改。
    • –res: apk里的资源文件和使用的id。
      • -drawable: apk的drawable资源，例如图片和自定义shape。
      • -layout: apk的布局文件。
      • -values: 里面有apk的颜色值、字符串、数组等定义文件，其中public.xml定义了app的所有自定义值的id，如果有新增加的值，必须要在public.xml上加上对应的id。
    • –smali: app的主要业务代码，使用的是smali语法（Dalvik虚拟机使用的汇编器），我们通常在这个目录下进行逆向开发。所以熟悉smali语法则是熟悉逆向开发的必要条件。具体语法这里就不介绍了，可以看http://mobile.51cto.com/aprogram-464015.htm。
  3. 打开AndroidManifest.xml，可以发现app的结构
    可以发现，Pedometer是App的启动Activity。我们在Smali目录中找到Pedometer.smali(注意，每一个类都有一个*.smali，这点和java的*.class类似)，现在我们找到onCreate方法，然后在调用setContentView方法后面加入按钮和点击事件。

    (代码都会在附录中给出）
    这里有一点需要注意，由于Android为了提高性能，把变量都存放在寄存器中，其中v表示本地寄存器，p表示参数寄存器，本地寄存器总数在方法第一行中声明。onCreate方法第一行.locals 14表示需要14个本地寄存器。最后再加入Pedometer$10.smali，这个类表示我们注入按钮的Onclick事件的匿名内部类。(注意，这些smali代码都不用自己写，可以使用工具把java转化成smali。）以上，我们已经成功的在原app上注入了我们的代码，现在只需在终端上输入java -jar apktool.jar b com.pedometers_220545.apk后回车，即可在/com.pedometers_220545/dist目录上找到com.pedometers_220545.apk。
  4. 此时如果我们安装这个apk就会发现，出现了Failure [INSTALL_PARSE_FAILED_NO_CERTIFICATES]错误
    这是因为Android程序都需要签名才能安装，所以我们需要对这个apk重新签名，这就需要用到开头下载的Auto-sign工具。进入Auto-sign目录，输入java -jar signapk.jar testkey.x509.pem testkey.pk8 om.pedometers_220545.apk com.pedometers_220545_signed.apk后回车（注意得把dist目录的apk放入Auto-sign下）就会生成已经签名好的com.pedometers_220545_signed.apk，这时候就可以成功安装了。

    （可以看到，我们已经成功注入了广告😂）
